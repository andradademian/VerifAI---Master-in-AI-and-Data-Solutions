# VerifAI — Fake News Credibility Analyzer

VerifAI is a web application that fetches live news articles and scores their
credibility with a fine-tuned **DistilBERT** classifier. Each analysis is paired
with a plain-language explanation generated by **Cohere**, so the reader gets
both a numeric signal and a human-readable "why".

This README documents the three components that make up the runnable product:

| Component | Path | Role |
|-----------|------|------|
| **Web application** | [`App/`](App/) | Flask backend + frontend that fetches news and serves predictions |
| **Model artifacts** | [`distilbert_frozen_hf/`](distilbert_frozen_hf/) | The classifier the app loads at runtime |
| **Training notebook** | [`DistilBERT/frozen_distilbert_rss (1).ipynb`](DistilBERT/frozen_distilbert_rss%20%281%29.ipynb) | Trains the **frozen** classifier the app uses + live RSS bias analysis |

> Other folders in the repo (`Data analysis/`, `FakeNewsModel/`, the remaining
> `DistilBERT/` notebooks, `pipeline_full.py`, etc.) are research/exploration
> artifacts and are **not** required to run the application.

## How it works

```
                 ┌─────────────────────────────────────────────┐
                 │                Browser (UI)                  │
                 │   templates/index.html + static/script.js    │
                 └───────────────┬───────────────┬──────────────┘
                                 │ GET           │ POST
                                 │ /api/articles │ /api/analyze
                 ┌───────────────▼───────────────▼──────────────┐
                 │                Flask app (App/app.py)         │
                 │                                               │
    newsdata.io ◄┤  /api/articles  → fetch & cache live news    │
                 │                                               │
                 │  /api/analyze   → DistilBERT inference        │──► distilbert_frozen_hf/
                 │                   + temperature-scaled probs  │
                 │                   + Cohere explanation        │──► Cohere API
                 └───────────────────────────────────────────────┘
```

1. The frontend loads the latest articles from `/api/articles` (proxied from
   [newsdata.io](https://newsdata.io)) with infinite scroll, search, and
   category filtering.
2. When a user analyzes an article, the frontend calls `/api/analyze`.
3. The backend tokenizes the text, runs the DistilBERT classifier, applies
   **temperature scaling** to avoid overconfidence, and derives margin-based
   confidence/uncertainty metrics.
4. Cohere turns the raw result into a short, friendly explanation of the
   article's *linguistic style* (no numbers repeated).

### API endpoints

| Method | Endpoint | Purpose |
|--------|----------|---------|
| `GET`  | `/` | Serves the main UI. |
| `GET`  | `/api/articles` | Fetches live news. Query params: `country`, `language`, `category`, `q` (search), `page` (pagination cursor). |
| `GET`  | `/api/article/<article_id>` | Returns a single cached article by id (`404` if not found). |
| `POST` | `/api/analyze` | Runs credibility analysis. Body: `{ "title", "text", "article_id" }`. |

**`/api/analyze` response schema:**

```json
{
  "status": "success",
  "article_id": "…",
  "analysis": {
    "label": "real | fake | uncertain",
    "classification": "Likely credible | Likely misinformation | Low confidence prediction",
    "prob_fake": 0.0,
    "prob_real": 0.0,
    "confidence": 0.0,
    "uncertainty": 0.0,
    "risk_score": 0.0,
    "decision_rule": "temperature-scaled softmax + margin-based uncertainty",
    "ai_recommendation": "…friendly Cohere explanation…",
    "demo_mode": false
  }
}
```

Key inference details (in `app.py`):

- **Input:** `"Title: {title}. Article: {text[:2000]}"`, lightly normalized to
  reduce topic/domain bias, truncated/padded to `MAX_LEN = 256` tokens.
- **Temperature scaling:** logits are divided by `TEMPERATURE = 2.0` before
  softmax to reduce overconfidence.
- **Uncertainty:** derived from the probability margin — an article is labelled
  `uncertain` when `uncertainty > 0.6`, otherwise `fake`/`real` by the larger
  probability.

## The model (`distilbert_frozen_hf/`)

This directory holds the classifier the app loads at startup
(`MODEL_DIR = <repo>/distilbert_frozen_hf`).

| File | In repo? | Description |
|------|----------|-------------|
| `config.json` | ✅ | DistilBERT architecture config: `DistilBertForSequenceClassification`, 6 layers, 12 heads, hidden dim 768, 2-label single-label classification, vocab 30522. |
| `tokenizer.json`, `tokenizer_config.json` | ✅ | Fast tokenizer for `distilbert-base-uncased`. |
| `model.safetensors` (weights) | ❌ **gitignored** | The trained weights. Produced by the frozen training notebook (see below). |

> **Why the weights aren't in git:** `*.safetensors`, `*.bin`, `*.pt`, and
> `*.npy` are excluded via `.gitignore` because they're large binaries. On a
> fresh checkout the weights file is therefore missing, and the app starts in
> [DEMO mode](#demo-mode--limitations). To run real inference, drop the trained
> weights into this folder.

`distilbert_frozen_hf` is the **frozen** / production variant used by the app —
the DistilBERT backbone is kept frozen and only a classification head is trained
(see [the training notebooks](#the-training-notebooks)). The sibling
`distilbert_lastlayer_hf/` directory is an alternative experiment (last layer
unfrozen) and is **not** used at runtime.

## The training notebooks

Two DistilBERT training notebooks live in `DistilBERT/`. They share the same
dataset and label convention but differ in **which parameters are trained** —
and only the *frozen* one produces the model the app ships with.

**Shared dataset:** [WELFake](https://www.kaggle.com/datasets/saurabhshahane/fake-news-classification)
(`WELFake_Dataset.csv`). The `title` and `text` columns are concatenated into a
single `content` field. Label convention: **`0 = Real`, `1 = Fake`**. Both
notebooks run **once on a GPU** (e.g. a Google Colab T4).

| Notebook | Approach | Trainable params | Produces |
|----------|----------|------------------|----------|
| [`frozen_distilbert_rss (1).ipynb`](DistilBERT/frozen_distilbert_rss%20%281%29.ipynb) | Backbone **frozen**, classification head only | ~1.5K | The **production** frozen model → `distilbert_frozen_hf/` |
| [`distilbert_1_training.ipynb`](DistilBERT/distilbert_1_training.ipynb) | **Full** fine-tuning (all layers) | ~66M | An alternative fully fine-tuned model (`distilbert_finetuned/`) — not used by the app |

### `frozen_distilbert_rss (1).ipynb` — production model

This notebook trains the classifier the app actually loads, and additionally
runs a live-news bias analysis.

**Training:**

1. Loads `distilbert-base-uncased` (`DistilBertModel`) and **freezes every
   transformer parameter** (`requires_grad = False`).
2. Adds a trainable head only: `Dropout(0.3) → Linear(768, 2)` on the `[CLS]`
   token — roughly **1,538 trainable parameters** out of ~66M.
3. Tokenizes with `MAX_LEN = 256`; trains with Adam (`LR = 2e-4`, higher because
   only the head learns), `BATCH_SIZE = 32`, `EPOCHS = 3`, 5% validation split
   (no test split).
4. Saves the best checkpoint by validation accuracy as a PyTorch state dict
   (`frozen_distilbert_best.pt`). This is later exported to Hugging Face format
   to become `distilbert_frozen_hf/`.

**Why frozen?** Keeping the Wikipedia/BookCorpus representations fixed stops the
model from overfitting to WELFake's publisher fingerprints and scraping
artifacts. It trades in-distribution accuracy for less biased, less overconfident
predictions on real-world news — which is exactly what the app needs.

**Live RSS bias analysis (second half of the notebook):**

- Fetches the last `HOURS_BACK = 24` hours of articles from **8 RSS feeds**
  (Reuters world/top, BBC world, Guardian world, AP, NYT world, Al Jazeera, NPR)
  via `feedparser`.
- Classifies each article and reports per-source P(fake) distributions, plots,
  and a ranked list, saving results to `frozen_distilbert_rss_<timestamp>.csv`.
- Purpose: check whether the frozen model gives more reasonable, less bimodal
  predictions on credible live sources than the fully fine-tuned model (which
  scored ~100% P(fake) on live news).

> **Output-column direction:** the notebook's inference helper returns a 2-column
> probability array and **auto-detects** which column is "fake" via a sanity
> check on known samples (`REAL_COL` / `FAKE_COL`). Keep this in mind when
> comparing against the app, which hardcodes `prob_fake = probs[0]` — verify the
> exported model's column order matches the app's assumption.

### `distilbert_1_training.ipynb` — full fine-tune (alternative)

Fine-tunes **all** DistilBERT layers: `distilbert-base-uncased`,
`num_labels=2`, `seq_classif_dropout=0.3`; stratified 70/15/15 split; AdamW
(`lr=2e-5`, weight decay `0.01` with no-decay groups), linear warmup (10%),
gradient clipping (`max_norm=1.0`), `BATCH_SIZE=16`, `NUM_EPOCHS=3`; best model
by weighted validation F1 saved via `save_pretrained`. It also writes
`test_texts.npy` / `test_labels.npy` / `training_history.npy` for the companion
evaluation notebook (`distilbert_2_evaluation.ipynb`). This model is **not**
loaded by the app.

**To wire a retrained model into the app:** export the trained weights to
Hugging Face format and copy `model.safetensors` (plus the tokenizer files if
changed) into `distilbert_frozen_hf/`.

## Setup & running

### Prerequisites

- Python 3.10+
- A [newsdata.io](https://newsdata.io) API key (required)
- A [Cohere](https://cohere.com) API key (optional — enables AI explanations)
- The trained `model.safetensors` in `distilbert_frozen_hf/` (optional — without
  it the app runs in DEMO mode)

### Install

```bash
cd App
python -m venv .venv
# Windows PowerShell:  .venv\Scripts\Activate.ps1
# macOS/Linux:         source .venv/bin/activate

pip install -r requirements.txt
# app.py also needs the ML runtime, which is not in requirements.txt:
pip install torch transformers
```

### Configure

Create `App/.env` (see [Configuration](#configuration-environment-variables)):

```env
NEWSDATA_API_KEY=your_newsdata_key
COHERE_API_KEY=your_cohere_key
```

### Run

```bash
cd App
python app.py
```

Then open **http://localhost:5000**. On startup the console reports whether the
AI model loaded successfully or the app is running in DEMO mode.

## Testing

The pytest suite stubs out `torch` and `transformers`, so it runs **without**
model weights installed.

```bash
cd App
pip install pytest
pytest tests/tests.py -v
```

It covers the `/api/analyze`, `/api/articles`, and `/api/article/<id>` endpoints,
including input validation, response schema, prompt-injection/XSS handling, and
error behavior.

`test_api.py` is a separate manual smoke test that hits the live newsdata.io API
and requires a real `NEWSDATA_API_KEY`.

## Demo mode & limitations

- **DEMO mode:** if `distilbert_frozen_hf/` has no weights file, the app still
  starts. `/api/analyze` then returns a deterministic placeholder score derived
  from a hash of the article text (`demo_mode: true`), and Cohere explanations
  still work. This keeps the UI and integration flow exercisable on a fresh
  checkout. Add the trained weights to switch to real inference.
- **News content:** articles whose content is `"ONLY AVAILABLE IN PAID PLANS"`
  (newsdata.io free tier) are filtered out or fall back to the description.
- **Not a fact-checker:** the classifier scores *linguistic style* and
  credibility signals, not ground-truth factual accuracy. Explanations
  deliberately avoid asserting an article is definitively "true" or "false".
